# v0.6 搜索导航与可视化

## 目标

把分散在知识库、图片库和记忆图谱中的内容汇聚成一个导航入口，并让图谱和统计数据真正可探索。

## 全局语义搜索

```text
查询文本
  → 百炼 Embedding（只计算一次）
  ├─ Elasticsearch document child
  ├─ Elasticsearch image chunk
  └─ Neo4j Statement embedding
  → 分类型绝对余弦门控
  → 文档 / 图片 / 记忆三列返回
```

- 文档和图片使用 Elasticsearch KNN，记忆在用户隔离的 Neo4j 陈述集合内计算余弦相似度。
- 默认最低相关度为 `0.25`，调用方可在 `0~1` 范围内调整。
- 文档按 `source_id` 去重，避免同一文档的多个子块占满结果。
- 返回内容携带目标类型、目标 ID、摘要、标签、相关度和来源元数据，可直接生成收藏快照。

## 收藏与标签

- 收藏支持 `document`、`image`、`memory` 和 `message`，以 `(user_id, target_type, target_id)` 保证幂等。
- 收藏保存轻量快照，即使原始内容后续改变，仍能解释收藏时看到的结果。
- 标签支持新建、重命名、换色和删除，并展示文档/图片使用数量。
- 标签名称发生变化时，通过 Elasticsearch `update_by_query` 同步所有当前用户的 chunk，避免数据库与搜索索引不一致。

## 每日回顾

每日回顾按上海时区汇总当天新增的：

- 用户提问；
- 主动或对话萃取的记忆来源；
- 文档；
- 图片。

有活动时由百炼生成不超过 180 字的中文简报，无活动时使用稳定兜底文案。结果按 `(user_id, review_date)` 持久化，可主动重新生成。

## 可视化

### AntV X6 记忆图谱

- 四列布局对应来源、片段、陈述和实体。
- 保留 `HAS_FRAGMENT`、`SUPPORTS`、`SUBJECT`、`OBJECT` 与实体关系边。
- 支持拖拽节点、画布平移，以及按住 Ctrl/Command 缩放。

### ECharts 仪表盘

- 实时统计文档、图片、记忆来源、记忆实体、收藏和标签数量。
- 折线图展示近 14 天记忆来源新增趋势。
- 环形图优先展示知识标签分布；没有标签时展示记忆社区分布。

X6 和 ECharts 所在页面使用 React 路由级懒加载，不阻塞认证和普通知识页面首屏。

## API

- `POST /api/search`：三路全局语义搜索与相关度门控。
- `GET|POST /api/favorites`、`DELETE /api/favorites/{id}`：收藏夹。
- `GET|POST /api/tags`、`PATCH|DELETE /api/tags/{id}`：标签管理。
- `GET /api/daily-review`：读取或重新生成每日回顾。
- `GET /api/dashboard`：统计、趋势和分布数据。

## 当前边界

- 全局入口优先保证语义门控，文档知识库页面仍保留 IK-BM25 + 向量 RRF + 可选 Rerank 的精细检索。
- 每日回顾目前按需生成；生产环境可再增加持久任务队列的每日定时触发。
- 全局搜索结果暂不直接打开原文定位页，后续可增加统一详情路由。
