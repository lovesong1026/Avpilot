# v0.5 可溯源长期记忆

## 目标

让 Avpilot 不只保存对话文本，而是把值得长期使用的信息变成结构化记忆；任何实体和陈述都能回到原始来源。

## 萃取入口

- 主动记住：`POST /api/memories` 提交一段文本，立即返回 PostgreSQL 审计任务。
- 对话萃取：一轮知识问答完成后，以独立协程分析用户消息，不阻塞 SSE 输出。
- 两种入口共用同一套萃取、去重、图谱写入和社区更新流程。

百炼模型只保留稳定画像、偏好、目标、人物关系和明确事件。低置信陈述、寒暄、普通问句和模型建议不会进入长期记忆。

## 四层溯源图谱

```text
MemorySource（主动记住 / 对话消息）
  └─ HAS_FRAGMENT → MemoryFragment（原文片段）
       └─ SUPPORTS → Statement（原子陈述）
            ├─ SUBJECT → Entity（主语）
            └─ OBJECT  → Entity（宾语）

Entity ─ RELATES_TO { predicate } → Entity
```

所有节点均带 `user_id`，查询和写入始终按用户隔离。PostgreSQL 的 `memory_sources` 保存原文、任务状态、错误和图谱统计，Neo4j 保存可导航的语义关系。

## 画像与事件

- `profile`：稳定身份、偏好、关注主题、目标和关系，用于用户画像。
- `event`：发生或计划发生的事件，保留 `event_time` 并进入时间线。
- 没有明确时间的事实不会被伪造日期。

## 两层去重

1. 规则层：实体按规范化名称复用，陈述按主语、谓词、宾语和事件日期组成的业务键复用。
2. 语义层：规则未命中时比较 Embedding；同类实体达到 `0.90`、同类陈述达到 `0.94` 才会复用。事件语义去重还要求日期一致。

写入共享陈述时会刷新其主语和宾语边，并清理失去溯源的孤立实体，防止图谱随重复萃取膨胀。

## 社区聚类

v0.5 使用实体关系图的连通分量完成轻量社区聚类，不要求安装 Neo4j GDS。每次萃取后重新计算当前用户社区，并把 `community_id` 回写实体节点。

## API

- `POST /api/memories`：主动记住并启动异步萃取。
- `GET /api/memories`：查看来源、状态和抽取统计。
- `GET /api/memories/graph`：读取四层图谱节点与边。
- `GET /api/memories/timeline`：读取去重后的事件时间线。
- `GET /api/memories/communities`：读取实体社区与成员。
- `DELETE /api/memories/{id}`：删除来源并清理失去溯源的图节点。

## 当前边界

- 异步任务当前运行在 FastAPI 进程内；生产部署时应换成 Redis/Celery 等持久任务队列。
- 社区名称当前取前三个实体名称，后续可增加 LLM 社区摘要。
- v0.5 页面先提供结构化星图卡片；AntV X6 可交互关系图在 v0.6 接入。
