# v0.4 多模态、网页与自动标签

## 目标

把知识输入从本地文档扩展到公开网页和图片，并让视觉内容像文本一样可以被自然语言检索。

## 图片闭环

```text
上传原图
  → 格式、体积与像素安全校验
  → 压缩副本送入百炼视觉模型
  → 生成描述 / OCR / 物体 / 场景
  → 聚合为可检索文本并生成向量
  → 写入 Elasticsearch 的 image chunk
  → 自然语言执行 BM25 + 向量 RRF 检索
```

原图保存在用户隔离的本地存储目录中。前端通过带 JWT 的内容接口读取图片，不暴露可以绕过鉴权的静态文件路径。

## 网页闭环

```text
提交公开 URL
  → DNS 与地址安全检查
  → 逐次校验重定向目标
  → 抓取 HTML 并抽取正文
  → 保存正文快照
  → 父子分块 / Embedding / ES 索引
```

网页抓取只允许公开 HTTP/HTTPS 地址，禁止访问环回地址、私有网络、链路本地地址和保留地址。响应必须是 HTML，正文响应上限为 5 MB。

## 自动标签

- 文档、网页和图片处理完成前调用百炼 Chat 模型进行宽泛主题分类。
- 分类提示词携带用户已有标签，要求优先复用。
- 数据库通过 `(user_id, normalized_name)` 唯一约束进行第二层去重。
- 单份内容最多关联两个 AI 标签；分类失败不阻断主入库流程。
- 标签名称同步写入 ES chunk，给后续全局搜索和标签筛选预留能力。

## API

- `POST /api/knowledge-bases/{id}/web-pages`：提交公开网页。
- `GET /api/images`：按知识库列出图片及处理状态。
- `POST /api/images`：上传图片并启动多模态分析。
- `POST /api/images/search`：执行图片语义混合检索。
- `GET /api/images/{id}`：读取图片语义档案。
- `GET /api/images/{id}/content`：鉴权读取原图。
- `DELETE /api/images/{id}`：删除原图、记录和检索索引。

## 当前边界

- 动态渲染、需要登录或强反爬的网页暂不使用无头浏览器抓取。
- GIF 以首帧进行视觉理解，原始动画文件仍完整保存。
- 自动标签当前展示在内容上，标签重命名、合并和独立管理界面留到搜索导航版本。
- 图片能够语义检索，但尚未作为附件加入聊天中的看图问答。
