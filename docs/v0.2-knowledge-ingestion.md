# v0.2 知识库检索闭环

## 目标

让登录用户完成“创建知识库 → 上传文档 → 自动处理 → 混合检索 → 查看来源”的最小闭环。

## 支持格式

- PDF（保留页码）
- Word `.docx`
- Markdown
- TXT
- HTML

单个文件最大 25 MB，可通过 `MAX_UPLOAD_SIZE_MB` 调整。

## 处理流水线

1. 文件按用户和文档 UUID 保存到本地 `storage/`。
2. PostgreSQL 记录文档与处理任务状态。
3. 解析器提取规范化文本与 PDF 页码。
4. 父块保留上下文，子块用于向量和 BM25 召回。
5. 百炼 `text-embedding-v4` 生成子块向量。
6. Elasticsearch 使用 IK 中文分词保存父子块和来源坐标。
7. 查询分别执行向量召回与 BM25 召回，通过 RRF 融合；可选 Rerank。
8. 命中子块后返回父块正文，并附文档名、页码或字符位置。

## API

- `GET /api/knowledge-bases`
- `POST /api/knowledge-bases`
- `DELETE /api/knowledge-bases/{id}`
- `GET /api/knowledge-bases/{id}/documents`
- `POST /api/knowledge-bases/{id}/documents`
- `DELETE /api/knowledge-bases/documents/{document_id}`
- `POST /api/knowledge-bases/{id}/search`

所有接口都要求 Bearer Token，并在 PostgreSQL 与 Elasticsearch 两侧按 `user_id` 隔离。
