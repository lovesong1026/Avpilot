# v0.7 智能 Agent Workflow

## 目标

把 v0.6 之前彼此独立的知识库、长期记忆、联网搜索和图片理解能力，接入同一条可观测、可追溯的智能问答流程。

## Agent 主链路

```text
用户问题与可选图片
        ↓
LangChain create_agent
        ↓
模型自主选择工具
        ├── knowledge_search
        ├── memory_search
        └── web_search（需用户允许）
        ↓
统一工具结果与引用
        ↓
百炼文本或视觉模型
        ↓
SSE 增量回答、工具轨迹与引用
```

## 双模式编排

- 默认使用 LangChain `create_agent` 驱动百炼模型的原生 Function Calling。
- 模型或兼容端点不支持工具调用时，自动降级为受限 ReAct JSON 循环。
- Agent 最多执行配置的步骤数，工具有独立超时，不允许调用注册表外的工具。
- 工具上下文在服务端注入 `user_id` 和知识库 ID，模型无法自行指定其他用户。

## 三类工具

### `knowledge_search`

- 复用 Elasticsearch IK-BM25、向量召回和 RRF 混合检索。
- 子块负责命中，父块提供完整上下文。
- 返回文档、片段、页码、字符位置和相关度。

### `memory_search`

- 查询当前用户 Neo4j 四层记忆图谱中的可搜索陈述。
- 使用 Embedding 绝对相似度门控。
- 返回来源、陈述类型、事件时间和原始记忆引用。

### `web_search`

- 只有用户打开“允许联网”后才注册到本轮 Agent。
- 使用百炼内置联网搜索，并规范化标题、URL 和摘要。
- 无可靠 URL 的结果不会进入引用列表。

## SSE 事件

在兼容原有事件的同时增加：

```text
agent_started
agent_fallback
tool_started
tool_completed
agent_completed
```

工具调用的名称、参数、状态、耗时、命中数和错误会写入助手消息的 `tool_calls` 字段，重新打开历史对话后仍可展示。

## 多模态问答

- 用户可以从自己的图片库选择最多三张已处理图片。
- 后端再次校验图片归属、状态和本地文件是否存在。
- Agent 规划阶段使用已入库的图片描述、OCR 和场景信息。
- 最终回答阶段把真实图片数据交给百炼视觉模型。
- 用户消息持久化图片 ID、文件名和受保护内容地址。

## 前端交互

- 实时展示“搜索知识库 / 搜索长期记忆 / 联网搜索”的执行状态。
- 支持“允许联网”开关，关闭时模型看不到联网工具。
- 支持选择图片附件，并在历史消息中恢复缩略图。
- 文档、记忆和网页引用使用同一引用抽屉；网页引用可跳转原始 URL。
- SSE 请求遇到 Access Token 过期时会刷新令牌并自动重试。

## 验收

- 后端单元测试覆盖工具注册边界、LangChain 工具适配、Function Calling 委托、ReAct 降级、统一引用和无来源引用约束。
- 真实百炼环境验证长期记忆工具选择与零命中处理。
- 真实百炼环境验证联网搜索、URL 引用与工具记录持久化。
- 真实浏览器验证工具轨迹、联网开关、图片选择、视觉回答和历史附件恢复。
