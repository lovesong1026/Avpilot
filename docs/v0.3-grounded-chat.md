# v0.3 可追溯流式问答

## 目标

把 v0.2 的知识库混合检索接入真实问答流程，使用户获得实时生成、能够回到原文核验的回答。

## 纵向闭环

```text
用户提问
  → 校验并保存会话与用户消息（PostgreSQL）
  → 对所选知识库执行 IK-BM25 + 向量 RRF 混合检索（Elasticsearch）
  → 将检索结果作为带编号的只读资料交给百炼模型
  → 通过 SSE 推送引用和回答增量
  → 保存助手消息与引用快照（PostgreSQL）
```

## API

### 会话

- `GET /api/conversations`：列出当前用户的会话。
- `POST /api/conversations`：创建会话并绑定知识库。
- `PATCH /api/conversations/{conversation_id}`：修改标题或所选知识库。
- `DELETE /api/conversations/{conversation_id}`：删除会话、消息和引用。
- `GET /api/conversations/{conversation_id}/messages`：读取消息与引用历史。

### 流式问答

`POST /api/chat/stream` 使用 `text/event-stream` 返回以下事件：

| 事件 | 作用 |
| --- | --- |
| `meta` | 返回会话 ID、标题和用户消息 ID。 |
| `retrieval_started` | 标记混合检索开始。 |
| `retrieval_completed` | 返回命中数量。 |
| `citation` | 一次返回本轮可核验的引用来源。 |
| `token` | 返回模型生成的文本增量。 |
| `completed` | 返回已持久化的助手消息 ID。 |
| `error` | 返回本轮失败原因。 |

请求示例：

```json
{
  "conversation_id": null,
  "message": "资料中的首次验收代号是什么？",
  "knowledge_base_ids": ["知识库 UUID"]
}
```

## 回答约束

- 系统提示词明确区分系统指令和检索资料，资料内容不能覆盖系统规则。
- 模型只能依据检索资料回答，并使用 `[1]`、`[2]` 标记事实来源。
- 检索无结果时不调用模型，直接返回“当前知识库中没有足够依据”。
- 引用保存为消息生成时的快照，后续查看历史时仍可核验当时使用的原文。

## 当前边界

- 当前流式生成与客户端连接同生命周期；客户端断开后不保证本轮继续执行。后续需要断线续传时，可将生成任务迁移到 Redis 队列并增加事件游标。
- v0.3 只编排知识库检索；记忆检索、联网搜索和完整 LangChain Agent 留到后续版本。
- 图片理解、网页入库和自动标签属于 v0.4。
